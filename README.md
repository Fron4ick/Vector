# Новогоднее веб‑шоу (основа)

## Запуск

1. Установите зависимости:

```bash
npm i
```

2. Запустите сервер:

```bash
npm run dev
```

Откроется:
- `http://localhost:5175/` — стартовая страница
- `http://localhost:5175/screen.html` — экран игры
- `http://localhost:5175/admin.html` — админка

## Сессии и доступ

Сессия одна (общая). Авторизации нет: админка и экран всегда связаны по умолчанию.

## Банки вопросов

Лежат в `data/packs/*.json`. Для каждого типа свой файл.

Схема (общая идея):
- `id` — уникальный id пака
- `title` — название пака
- `type` — тип (`guess_melody`, `lyrics_synonyms`, `emoji_song`)
- `questions[]` — массив вопросов

У вопроса можно указать:
- `title`, `prompt`, `hint`, `answer`
- `media.audio` — путь к аудио в `public/` (например `/media/last_christmas.mp3`)
- `media.video` — путь к видео в `public/` (например `/media/fragment.mp4`)
- `media.images[]` — массив путей к картинкам (например `/media/q1_1.jpg`)
- `media.autoPlayOnAnswer` — если `true`, то на экране ТВ медиа будет пытаться запуститься автоматически при открытии ответа

## Таймер и эффекты

В админке доступны:
- запуск/остановка таймера (в секундах) — отображается на экране ТВ
- эффекты: «Дробь» и «Динь» (синхронно проигрываются на экране ТВ)

## Медиа

Файлы кладите в `public/media/`.

Пример:
- `public/media/track1.mp3`
- в вопросе: `"audio": "/media/track1.mp3"`

## Примечание

Сейчас реализована базовая синхронизация состояния и минимальный набор типов с несколькими демо‑вопросами.
